# ГРОСС Бережливый Монтаж: roadmap

## Завершено

- [x] Брендинг и визуальный MVP.
- [x] Иерархия `Объект -> Корпус -> Этаж -> Дверь`.
- [x] Роли и role-based navigation.
- [x] Supabase/PostgreSQL схема и RLS.
- [x] Supabase Auth, приглашения и управление аккаунтами.
- [x] Приватное файловое хранилище.
- [x] Двери, проёмы, замечания ТН и акты ОХ.
- [x] Задачи, автоматические просрочки и уведомления.
- [x] Центр проблем и управленческий дашборд.
- [x] Бригады, нормы, план-факт и рабочая сила.
- [x] Базовый финансовый контур.
- [x] Аудит, Sentry-интеграция и security hardening.
- [x] CI с чистым database reset, lint и pgTAP/RLS-тестами.
- [x] Staging/production workflows, backup и restore drill.
- [x] Транзакционные workflow двери, акта, документа задачи и иерархии.

## В работе: пилотная готовность

- [ ] Завершить декомпозицию `LegacyApp` на независимые страницы и доменные модули.
  Экран входа, восстановление пароля, список и управление объектом/корпусами, визуализация корпуса, план этажа, маркеры, рабочая карточка двери, управление аккаунтами, личный кабинет и админ-конструктор типового этажа уже вынесены в автономные модули. Компонентные тесты фиксируют права ИТР/руководителя, фактическое количество этажей без demo-дверей, генерацию и применение шаблона ко всем этажам, приватную подложку плана, переходы по уровню и двери, фильтры плана, визуальные приоритеты статусов, приглашение аккаунтов через backend, подтверждённую смену пароля, загрузку приватного аватара и единый workflow сохранения двери/акта.
- [ ] Убрать оставшиеся demo-вычисления из production-пути.
- [ ] Расширить component/E2E-тесты на все ключевые роли и пользовательские маршруты.
  Базовый E2E маршрута ИТР с сохранением статуса и browser-проверка доступа четырёх ролей уже включены в CI.
- [ ] Провести performance-аудит больших объектов и списков.
- [x] Настроить реальные staging Supabase и Vercel, включая канонический alias и release smoke.
- [ ] Настроить и проверить staging Sentry с тестовым событием и alert contact.
- [x] Создать четыре staging-аккаунта и выполнить auth smoke.
- [ ] Импортировать пилотный объект и сверить контрольные количества.
- [ ] Провести UAT с руководителем и ИТР.
