# ГРОСС Бережливый Монтаж: roadmap разработки

## Этап 0. Старт проекта

- Создать базовую структуру репозитория.
- Описать назначение проекта.
- Зафиксировать краткое техническое задание.
- Разложить развитие по этапам.

## Этап 1. Архитектура MVP

- Выбрать технологический стек.
- Описать модель данных.
- Спроектировать роли и права доступа.
- Определить формат хранения файлов и фото для актов ответственного хранения при необходимости.
- Подготовить визуальный UI-флоу закрытой части.
- Зафиксировать брендинг ГРОСС Бережливый Монтаж.
- Зафиксировать, что сайт-визитка компании разрабатывается после MVP.

## Этап 2. Авторизация

- Реализовать вход по логину и паролю.
- Добавить хранение пользователей.
- Добавить защищенные маршруты.
- Добавить выход из системы.

## Этап 3. Реестр объектов

- Реализовать список объектов.
- Добавить карточку объекта.
- Добавить корпуса внутри объекта.
- Добавить базовые операции создания и редактирования.

## Этап 4. Структура объекта

- Реализовать этажи внутри корпусов.
- Реализовать навигацию по иерархии.
- Подготовить быстрый переход к дверям этажа.
- Подготовить конструктор объекта для создания корпусов, этажей и дверей.

## Этап 5. Двери и статусы

- Реализовать список дверей на этаже.
- Добавить карточку двери.
- Добавить изменение статуса двери.
- Добавить историю изменений статуса.

## Этап 6. Проемы и замечания

- Добавить учет состояния проемов.
- Добавить создание и закрытие замечаний.
- Добавить фильтрацию по статусам.
- Добавить журнал замечаний по объекту.

## Этап 7. Акты ответственного хранения

- Добавить карточку акта.
- Связать акт с дверью или группой дверей.
- Добавить загрузку файлов или фото для актов при необходимости.
- Добавить список актов по объекту.

## Этап 8. Отчеты MVP

- Добавить сводку по объекту.
- Добавить счетчики дверей по статусам.
- Добавить выгрузку базовых данных.
- Подготовить основу для расширенных отчетов.

## Этап 9. Подготовка к эксплуатации

- Настроить окружения разработки и продакшена.
- Добавить резервное копирование данных.
- Добавить базовое логирование.
- Подготовить инструкцию администратора.
- Провести пилот на одном объекте.

## Этап 10. Сайт-визитка после MVP

- Спроектировать публичную часть сайта компании.
- Подготовить структуру страниц и контент.
- Определить связь публичного сайта с закрытой системой, если она потребуется.
- Реализовать сайт-визитку отдельным этапом после стабилизации MVP.
