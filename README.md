# ГРОСС Бережливый Монтаж

ГРОСС Бережливый Монтаж - MVP внутренней системы компании ГРОСС для цифрового управления монтажом дверей на строительных объектах.

MVP фокусируется на закрытой части системы: авторизация, визуальная навигация по строительной структуре объекта и учет состояния дверей, проемов, замечаний и актов ответственного хранения.

Публичный сайт-визитка компании не входит в MVP и будет добавлен позже отдельным этапом.

## Цель MVP

Сделать рабочую основу системы, в которой сотрудники смогут:

- войти по логину и паролю;
- видеть список объектов;
- переходить по визуальной иерархии объект -> корпус -> этаж -> дверь;
- просматривать и менять статусы дверей;
- вести учет проемов;
- фиксировать замечания;
- хранить данные по актам ответственного хранения;
- прикладывать файлы или фото только к актам ответственного хранения, если это необходимо по процессу.

## Запуск MVP-прототипа

Для запуска нужен Node.js.

```bash
npm install
npm run dev
```

После запуска Vite покажет локальный адрес приложения, обычно `http://127.0.0.1:5173/`.

Для проверки production-сборки:

```bash
npm run build
```

В прототипе используется простая мок-авторизация.

```text
Логин: admin
Пароль: 123456
```

## Предполагаемые модули

- Авторизация и роли пользователей.
- Реестр объектов.
- Иерархия строительства: корпуса, этажи, двери.
- Реестр дверей и их статусы.
- Учет проемов.
- Замечания и история их устранения.
- Акты ответственного хранения.
- Отчеты и выгрузки.

## Конструктор объекта

Конструктор объекта должен создавать структуру:

```text
Объект
  Корпуса
    Этажи
      Двери
```

## Ограничения MVP

- Система является закрытой внутренней частью без публичного сайта.
- Файлы и фото используются только для актов ответственного хранения при необходимости.
- Сайт-визитка компании будет спроектирован и добавлен после запуска MVP.

## Статус проекта
