# Схема данных

Основная иерархия:

`companies -> objects -> buildings -> floors -> doors`

Пользователь Auth имеет профиль `profiles` с ролью и `company_id`. Таблицы
`object_assignments` и `building_assignments` определяют операционный доступ
директоров и ИТР. Привязки через массивы UUID больше не используются.

## Границы ответственности

- `companies`: юридическая/организационная область данных.
- `profiles`: профиль пользователя, связанный с `auth.users` один к одному.
- `objects`: строительные объекты компании.
- `buildings`: корпуса с фактическим количеством этажей.
- `floors`: этажи и снимок применённого шаблона.
- `doors`: единая сущность двери, её статусы, фактические размеры и контрольные даты.
- `object_assignments`: назначение пользователя на объект.
- `building_assignments`: назначение пользователя на корпус.

Идентификатор двери стабилен и далее используется задачами, документами,
актами, замечаниями, историей и отчётами.
