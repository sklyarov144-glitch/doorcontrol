# Подключение пользователей

## Порядок

1. Руководитель утверждает ФИО, рабочий email, телефон, роль и область доступа.
2. Уполномоченный пользователь отправляет приглашение через Edge Function `invite-user`.
3. Пользователь задаёт уникальный пароль по защищённой ссылке.
4. Администратор назначает объект/корпус; доступ проверяется отдельной сессией пользователя.
5. Пользователь проходит 20-минутное обучение и выполняет контрольный сценарий.

Нельзя создавать общие аккаунты «ИТР объекта» или передавать пароли в мессенджере. При увольнении профиль отключается в тот же рабочий день, сессии отзываются, объекты переназначаются.

Первый аккаунт среды создаётся только операционной командой
`npm run auth:bootstrap`. Команда одноразовая и блокируется после появления первого
профиля. Повторный запуск или создание пользователей напрямую через Auth Dashboard
не является штатным процессом.

## Минимальное обучение ИТР

- вход и личный кабинет;
- объект → корпус → этаж → дверь;
- изменение рабочего статуса;
- задачи, комментарии и документы;
- уведомления и закрытие акта ОХ;
- куда сообщать об ошибке или неверных данных.

## Проверка пилотной команды

После того как приглашённые пользователи приняли приглашения, заполните копию
`pilot/users.template.json` реальными рабочими email и выполните:

```bash
SUPABASE_URL=https://PROJECT.supabase.co \
SUPABASE_SERVICE_ROLE_KEY=... \
SUPABASE_COMPANY_ID=... \
npm run pilot:users -- /absolute/path/pilot-users.json
```

Команда требует руководителя компании, директора и минимум двух ИТР, проверяет
активный профиль, компанию и роль. Результат `pilot/user-assignments.json`
содержит только ключи и UUID для полей ответственных import-файла; он игнорируется
Git и не содержит email, паролей или service key.
