# Приёмочный чек-лист (UAT)

Каждый пункт фиксируется как Pass/Fail с датой, пользователем и ссылкой на дефект.

## Общие сценарии

- вход, выход, восстановление пароля и повторный вход работают;
- неактивный пользователь не получает доступ;
- прямой URL после обновления страницы открывается;
- sidebar сворачивается, основные страницы работают на ноутбуке;
- ошибки отображаются понятно и не раскрывают технические секреты.

## Роли и RLS

- creator видит разрешённый глобальный контур;
- company_head видит только свою компанию;
- construction_director видит назначенные объекты;
- ИТР видит назначенные объекты/корпуса и не открывает admin/finance;
- пользователь компании A не читает и не изменяет данные компании B;
- недоступные операции блокируются БД, а не только скрываются в UI.

## ИТР-маршрут

- Мои объекты → объект → корпус → этаж → план → дверь;
- изменение статусов сохраняется после перезагрузки;
- даты монтажа, ТН и акта устанавливаются корректно;
- цвет двери соответствует приоритетному статусу;
- задача, комментарий, ссылка и документ доступны согласно роли.

## Руководитель

- dashboard считает показатели из production provider;
- Центр проблем и задачи открывают правильный контекст;
- ответственные назначаются и видят свои объекты;
- финансовые показатели недоступны ИТР;
- отчёты совпадают с контрольной выборкой исходных данных.

## Файлы и аудит

- приватный файл нельзя открыть без авторизации;
- signed URL имеет ограниченный срок;
- история фиксирует критические изменения и пользователя;
- удаление/отключение пользователя не уничтожает историю.

## Производительность

- стартовая страница открывается без ошибок;
- p95 публичного smoke ниже 2 секунд на согласованном окружении;
- план с пилотным числом дверей остаётся отзывчивым;
- 20 параллельных пилотных пользователей не вызывают ошибок/лимитов.
