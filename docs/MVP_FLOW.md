# ГРОСС Монтаж: путь пользователя MVP

Документ описывает основной пользовательский путь в закрытой части системы.

## 1. Вход

Пользователь открывает ГРОСС Монтаж и видит форму входа.

Поля формы:

- логин;
- пароль.

После успешного входа пользователь попадает в список доступных объектов. Если данные неверные, система показывает ошибку входа.

## 2. Список объектов

На экране отображаются строительные объекты, доступные пользователю.

Для каждого объекта показываются базовые данные:

- название;
- адрес или краткое описание расположения;
- статус объекта;
- ответственный сотрудник, если назначен.

Пользователь выбирает объект и переходит внутрь его структуры.

## 3. Корпус

Внутри объекта пользователь видит список корпусов.

Корпус является первым уровнем детализации объекта. Пользователь выбирает нужный корпус, чтобы перейти к этажам.

## 4. Этаж

Внутри корпуса пользователь видит список этажей.

Этаж является уровнем, на котором открывается список дверей. Пользователь выбирает этаж и переходит к дверям этого этажа.

## 5. Список дверей

На экране этажа отображается список дверей.

Для каждой двери показываются:

- номер или маркировка;
- тип двери, если указан;
- текущий статус двери;
- состояние проема;
- наличие открытых замечаний;
- наличие связанного акта ответственного хранения.

Пользователь может открыть карточку конкретной двери.

## 6. Карточка двери

Карточка двери собирает основную информацию по выбранной двери.

В карточке отображаются:

- объект, корпус и этаж;
- номер или маркировка двери;
- тип двери;
- статус двери;
- данные по проему;
- замечания;
- акты ответственного хранения;
- комментарий или служебные заметки.

Пользователь может обновить статус двери, состояние проема, создать или закрыть замечание, а также связать дверь с актом ответственного хранения.

Файлы и фото в MVP используются только для актов ответственного хранения, если это необходимо.

## Конструктор объекта

Конструктор объекта в MVP должен создавать структуру:

```text
Объект
  Корпуса
    Этажи
      Двери
```
