# Роли и разрешения

## Роли

- `creator` — системный владелец, полный доступ.
- `company_head` — руководитель компании, управление компанией и объектами.
- `construction_director` — управление назначенными объектами и сотрудниками ИТР.
- `itr` — рабочие действия на назначенных объектах и корпусах.

## Базовая матрица

| Действие | creator | company_head | construction_director | itr |
| --- | --- | --- | --- | --- |
| Просмотр назначенных объектов | Да | Да | Да | Да |
| Создание объекта | Да | Да | Нет | Нет |
| Управление корпусами и этажами | Да | Да | Да | На назначенных объектах |
| Изменение рабочих статусов двери | Да | Да | Да | Да |
| Постановка задач | Да | Да | Да | Нет |
| Управление пользователями | Да | Да | ИТР своих объектов | Нет |
| Админ-панель | Да | Да | Да | Нет |
| Дашборд компании | Да | Да | Свои объекты | Нет |

Frontend использует функции `canView`, `canCreate`, `canEdit`, `canDelete`, `canManageUsers` и `canManageObjects` из `domain/permissions.js`.

Frontend-проверки нужны для интерфейса, но не являются защитой данных. После подключения Supabase ограничения должны быть продублированы политиками Row Level Security. Серверная политика всегда имеет приоритет над видимостью кнопок и меню.
