# ГРОСС Бережливый Монтаж

## Цель продукта

`ГРОСС Бережливый Монтаж` — внутренняя цифровая система управления компанией по монтажу дверей.

Руководитель за 30 секунд понимает состояние компании.  
ИТР закрывает рабочий день за 5 минут.  
Бригады становятся прозрачными по плану, факту, деньгам и качеству.

## Основные контуры

1. Объекты и двери
2. Документы
3. Задачи и уведомления
4. Акты ОХ / Замечания ТН / Центр проблем
5. План бригад / План-факт
6. Дашборд руководителя
7. Технический фундамент

## Роли

- `creator` — создатель системы, видит все разделы и настройки.
- `company_head` — руководитель компании, видит дашборд, деньги, риски, задачи, документы и пользователей.
- `construction_director` — директор строительства, управляет своими объектами, задачами, актами, проблемами и план-фактом.
- `itr` — инженер ИТР, работает с задачами, объектами, документами, планом бригад и уведомлениями.

## Что уже есть

- Ролевая навигация.
- Объекты, корпуса, этажи и карточки дверей.
- Визуальный путь ИТР: объект -> корпус -> этаж -> план -> дверь.
- Документы и шахматки как ссылки на Яндекс.Диск.
- Ручные задачи от руководителя.
- Автоматические задачи по просрочкам после монтажа.
- Центр уведомлений.
- Центр проблем.
- Контроль актов ОХ.
- Раздел замечаний ТН.
- Дашборд руководителя.
- План бригад, ежедневный факт и план-факт.
- Денежные показатели, рейтинг бригад, рейтинг ИТР и автоотчёт за день.

## В разработке

- Более глубокая связка всех документов с объектами, корпусами, этажами и дверями.
- Расширенная аналитика по ответственным и причинам отставаний.
- Улучшение демо-данных для продающего сценария.
- Подготовка к backend, базе данных и настоящей авторизации.

## Что не делаем сейчас

- Backend.
- Реальную базу данных.
- Серверную авторизацию.
- Загрузку файлов внутрь сайта.
- Интеграцию с API Яндекс.Диска.
- Полноценную шахматку внутри сайта.
- Отдельное мобильное приложение.

## Дорожная карта

- Этап 2 — быстрый контур ИТР.

## Как запустить

```bash
npm install
npm run dev
```

Открыть локальный адрес Vite, обычно:

```text
http://localhost:5173/
```

## Демо-роли

Вход в MVP: `admin`.

После входа можно переключать пользователя в верхней панели:

- Иван — `creator`
- Руководитель — `company_head`
- Директор строительства — `construction_director`
- ИТР — `itr`

## Сценарий для руководителя

1. Открыть `Дашборд`.
2. Посмотреть готовность, риски, акты, замечания, деньги и проблемные корпуса.
3. Открыть `Центр проблем`.
4. Поставить задачу ИТР.
5. Открыть `План бригад`.
6. Посмотреть план-факт, деньги, рейтинг бригад и автоотчёт.

## Сценарий для ИТР

1. Открыть `Мои задачи`.
2. Взять задачу в работу.
3. Добавить комментарий или ссылку на Яндекс.Диск.
4. Открыть дверь из задачи.
5. Изменить статус двери крупными рабочими действиями.
6. Внести факт за день в `План бригад`.
7. Сформировать автоотчёт.
